## 📖 مقدمة عن المشروع / Overview

**المعلم العربي المحترف** هو تطبيق ويب متكامل (Full-Stack) صُمم خصيصاً لتمكين المعلمين والمعلمات في الوطن العربي من إعداد خطط الدرس النموذجية، وتوليد الوحدات التعليمية المتكاملة، وتلخيص المستندات المنهجية الكبرى ومناقشتها تفاعلياً بشكل آمن ومؤصل تربوياً.

التطبيق يتبنى فلسفة **الكاتب والمفكر وضاح الظليل (Waddah Al-Zulail)** التي تدعو إلى **الديتوكس الرقمي والحد من استخدام الهاتف** والعودة إلى الواقع الملموس؛ حيث تدمج المكونات التربوية نصائح وجدانية عميقة تحث على الهدوء الفكري، والتعلم النشط، والوسائل التعليمية الحسية بعيداً عن تشتيت الشاشات المستمر.

## 🛠️ البنية التقنية للمشروع / Technical Stack

التطبيق مبني بنظام **Full-Stack (Client + Server)** لضمان الحماية الكاملة لمفاتيح الأمان وسرعة الأداء:

1. **الواجهة الأمامية (Frontend):**
   * **React 19** بالتعاون مع **Vite** للتطوير والترجمة فائقة السرعة.
   * **Tailwind CSS (v4)** لتنسيق الواجهات بمرونة تامة دون ملفات CSS مكررة.
   * **Motion (from `motion/react`)** لإضافة حركات انتقالية ناعمة ومحفزة بصرياً للواجهة.
   * **pptxgenjs** لتصدير خطط الدروس والشرائح إلى ملفات PowerPoint تفاعلية جاهزة للتحميل والعرض المباشر.

2. **الواجهة الخلفية (Backend - Express Server):**
   * خادم **Express.ts** يعمل كـ Proxy آمن للتعامل مع الذكاء الاصطناعي من جوجل.
   * **Google GenAI SDK (`@google/genai`)** المحدث للاتصال بنموذج الذكاء الاصطناعي السريع والدقيق `gemini-3.5-flash` مع ضبط المخرجات بنظام Schema وهيكلية JSON صارمة لمنع انهيار الواجهة.
   * مكتبات معالجة المستندات المرفوعة (`pdf-parse`, `mammoth` لملفات DOCX، وقارئ النصوص المباشرة) لتغذية وحدة التحليل والتلخيص الفوري.

## ⚠️ مشكلة النشر على Vercel وحلها الجذري / Vercel Deployment Issue Explained

### 🔍 سبب المشكلة / Root Cause
عند نشر مشروع Full-Stack يعتمد على خادم Express مستمر للـ API (مثل `server.ts`) على منصة **Vercel** بشكل افتراضي بدون إعدادات خاصة، تظهر مشكلة:
`Unexpected token '<', "<!doctype "... is not valid JSON`

* **التفسير التقني:** منصة Vercel مخصصة افتراضياً لاستضافة التطبيقات الساكنة (Static Apps / SPAs). عندما يحاول المتصفح إرسال طلب `POST` إلى `/api/gemini/lesson-plan` لتوليد الدرس، لا يجد خادم Express نشطاً في الخلفية لاستقبال الطلب.
* بدلاً من ذلك، يقوم موجه Vercel الساكن بإرجاع ملف الواجهة الرئيسي `index.html` (والذي يبدأ بـ `<!doctype html>`) كحل بديل للطلبات غير المعروفة. يحاول كود المتصفح تحليل هذا النص كـ JSON فيفشل وتظهر الرسالة المذكورة وتتوقف عجلة التوليد.

### 🚀 الحلول المقترحة للنشر بنجاح / How to Deploy Correctly

لحل هذه المشكلة وجعل التطبيق يعمل بكفاءة تامة خارج بيئة Google AI Studio، يمكنك اتباع أحد الحلول التالية:

#### 💡 الحل الأول: النشر على منصات تدعم خوادم Node.js الكاملة (موصى به جداً)
بما أن التطبيق يملك خادم Express متكامل وجاهز للتشغيل والإنتاج عبر الأوامر الموجودة في `package.json` (`npm run build` ثم `npm start`)، فإن أفضل منصات لاستضافته مجاناً وبأعلى كفاءة هي المنصات المخصصة للحاويات وخوادم الويب المستمرة:

1. **منصة Railway (سهل وسريع):**
   * اربط حسابك بـ GitHub.
   * اختر مستودع المشروع وسيقوم Railway تلقائياً بالتعرف على ملف `package.json` وبناء المشروع وتشغيل خادم Express على المنفذ الصحيح.
   * أضف المتغير البيئي `GEMINI_API_KEY` في إعدادات البيئة (Variables) على منصة Railway.

2. **منصة Render:**
   * أنشئ خدمة ويب جديدة (Web Service) واربطها بالمستودع.
   * اضبط أمر البناء (Build Command): `npm install && npm run build`
   * اضبط أمر التشغيل (Start Command): `npm start`
   * أضف المتغير البيئي `GEMINI_API_KEY` في قسم الإعدادات المتقدمة (Advanced / Environment Variables).

---

#### 💡 الحل الثاني: النشر على Vercel باستخدام الدوال اللامركزية (Vercel Serverless Functions)
إذا كنت تفضل بشدة البقاء على منصة Vercel، يجب عليك تحويل خادم Express إلى دالة برمجية تفاعلية (Serverless Function). 

1. أنشئ مجلداً جديداً في جذر المشروع باسم `api`.
2. أنشئ ملفاً بداخله باسم `index.ts` واستورد خادم Express الخاص بك ليكون واجهة الاستقبال الأساسية لـ Vercel:
```typescript
// api/index.ts
import app from "../server"; // تأكد من تصدير المتغير app من ملف server.ts بدلاً من تشغيل listen مباشرة عند الاستيراد في بيئة السيرفرليس
export default app;
```
3. قم بإنشاء ملف `vercel.json` في جذر المشروع لتوجيه مسارات الـ API إلى الدالة الجديدة:
```json
{
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api" },
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

## 💻 التشغيل المحلي والتطوير / Local Development

لتشغيل المشروع محلياً على جهازك الشخصي بكل سهولة:

1. **تثبيت الملحقات:**
   ```bash
   npm install
   ```

2. **تهيئة المتغيرات البيئية:**
   قم بإنشاء ملف باسم `.env` في المجلد الرئيسي للمشروع، وأضف مفتاح الأمان الخاص بك:
   ```env
   GEMINI_API_KEY=your_actual_gemini_api_key_here
   ```

3. **بدء التشغيل في بيئة التطوير:**
   ```bash
   npm run dev
   ```
   سيقوم التطبيق بتشغيل خادم Express الفوري ودمج واجهة Vite عليه، لتتمكن من تصفح التطبيق على الرابط المحلي: `http://localhost:3000`

4. **البناء والإنتاج الفعلي:**
   ```bash
   npm run build
   npm start
   ```

## 📁 هيكلية المجلدات الأساسية / Folder Structure

* `server.ts` - ملف الخادم الرئيسي المتحكم في تدفق الـ API والاتصال بنموذج Gemini ومعالجة الملفات.
* `src/App.tsx` - الواجهة الرائدة للمنصة والمتحكمة في الانتقال بين التبويبات والمستشار التربوي المساعد.
* `src/components/` - المكونات الفرعية التفاعلية (دفتر الدرجات، المستشار الصوتي، تحضير الدروس الذكي، تلخيص الكتب المدرسية تفاعلياً).
* `src/data/curriculumData.ts` - قاعدة البيانات الثابتة للمناهج العربية واليمنية المحملة مسبقاً لضمان الدقة وتفادي الهلوسة البرمجية.
